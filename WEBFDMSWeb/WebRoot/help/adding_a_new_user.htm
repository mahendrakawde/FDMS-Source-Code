Adding a New User

Use this function to create a new user and enter all their user information as well as their user privileges.

  1. While viewing the User Administration dialog box, click the Add button. The User Administration Add window displays.

  2. Login Name.  Enter the login name for this user. This selection is used in the "User Name" field when logging into the system. The Login Name must be unique.

  3. Password.  Enter a password to associate with the Login Name. This selection is used in the "Password" field when logging into the system.

  4. First Name.  Enter the user’s first name.

  5. Email.  Enter the user’s email address.

  6. Confirm Password.  Reenter the password entered in the Password field, above.

  7. Last Name.  Enter the user’s last name.

  8. Initials.  This feature is not currently used.

  9. Under the User Privileges, click in the check box next to each privilege to grant to this user based upon the following guidelines:

  10. Administrator.  Restricts access to Admin screen (Edit Price List, User Administration, etc.).

  11. At-Need Arrangements.  Restricts access to Call Info, Vitals, Services, Survivors, Obit, Vets, Custom and Donations screens.  Also prevents a user from entering a new At-need case.

  12. Pre-Need Arrangements.  Restricts access to pre-need screen.  Also, prevents user from entering a new pre-need case.

  13. Financial Contract Data.  Restricts access to the Financial Information page.

  14. Payments on Accounts.  Restricts access to the Payment screen.

  15. View Only.  User cannot save on any screen they are allowed access to.

  16. Accts Receivable Processing.  Restricts access to the Financial tab (Finance Charges, Check Writer, Misc. Cash Receipts, Financial Reports, and Accounting Interfaces).

  17. Print Forms.  Restricts access to the Forms Completed box on the Status window so the user cannot print forms such as the Death Certificate, Invoice Letter, etc.

  18. Print Reports.  Removes Reports tab so user cannot print At-need or Pre-need reports.

  19. Void Cases.  User cannot void a case if this privilege is not granted.

  20. Inventory.  Removes Inventory tab so user cannot view catalog, receive inventory items, etc.

  1. When all selections/entries are made, click Save.
